If you are sending a message to someone, you expect them, and only them, to see your message, know it's from you (in most cases), and have the message be as you sent it. If you are someone dealing with sensitive information, this is especially important. Now, messages and records and almost all information are stored and sent digitally. And if you deal with this information, you expect for the tools that you use to be secure. As someone who sends texts and emails, I take for granted that my message is being safely delivered. But computers carry inherent traits that leak information. In simpler terms, someone dedicated to stealing this digital information has avenues that people rarely consider. These sorts of attacks are called Side Channel Attacks. When you imagine a hacker, you imagine them using software or viruses to try and break into a computer; side channel attacks, on the other hand, are very hardware oriented, and are very powerful. Software attacks can be prevented by robust software and intelligent users (although this is not something that you should expect). However, Side channel attacks use hardware characteristics of a computer to steal information. To get a better understanding of how some side channel attacks work, first we need to talk about encryption and cryptography. The simplest way to understand it is that a message you send is getting turned into something else. This process is called encryption. Then you send the message, which cannot be interpreted in its “encrypted” state. Someone who receives the message then has a process where they turn the encrypted message back into the original. Traditional side channel attacks take advantage of the process where a message is encrypted. This is because a computer’s hardware state can vary based on the message. Without getting into too much detail about how a computer works: a computer will unintentionally leak information while doing its tasks. Some good examples of this are power, time, and electromagnetic waves. The most widely known is a Power Side Channel Attack. The basic idea of this is that a computer needs power to do its tasks. If an attacker can measure the power consumption of a computer, they can use that information to learn secrets. During the process when a computer is encrypting a message, it uses power. If the power used depends on the message that is being encrypted, then that power consumption leaks information. A simple power side channel attack uses a graph of the power consumption to directly determine what operations the computer is doing. Encryption algorithms do certain things (imagine additions and multiplication), and an attacker can map each pattern on the graph to an operation being done. This can leak how the encryption algorithm works, and the attacker can then use this information to unencrypt a message. Of course, the power consumption will not be a nice graph, especially for modern computers. Especially because modern computers can do many things at the same time, it is not very easy for a simple mapping between a graph and operations to be made. However, it is still possible. More complex methods called Differential and Correlated power analysis can be used. The details of how these work are too much for this blog, but can be found here. The basic idea, for those with some statistical knowledge, is to get multiple graphs of the power consumption. From here, the mean can be computer, giving a graph that reduces the noise that comes from other sources. With modern machine learning models, Some mitigation strategies for a power side channel attack is to simply make the power consumption not vary. (easier said than done).
